# CLAUDE.md — Одностраничник распива парфюмерии (ИП Жданов)

Статический сайт-витрина (landing) для распива нишевой/люксовой парфюмерии: отливанты
**3 / 4 / 5 мл**, каталог из 23 ароматов, корзина и оформление заказа без онлайн-кассы.
Цель — показать реальную онлайн-деятельность ИП. Товар до 5 мл не подлежит обязательной
маркировке «Честный знак» и сертификации.

## Стек
Чистый **vanilla HTML/CSS/JS, без сборки и бэкенда**. Хостится где угодно (Netlify/Vercel/
GitHub Pages/Beget). Данные корзины — в `localStorage`. Заказ уходит заявкой в мессенджер/почту.

## Структура
```
index.html              витрина: промо-плашка, шапка, hero, лента брендов + строка доверия,
                        каталог, оплата/доставка, футер; разметка корзины и модалки заказа
offer.html              публичная оферта
privacy.html            политика конфиденциальности (152-ФЗ)
assets/css/styles.css   ВСЕ стили (светлая «персиковая премиум» тема)
assets/js/config.js     ← КОНТАКТЫ, РЕКВИЗИТЫ ИП, КАНАЛЫ ЗАКАЗА, СКИДКА, НАЗВАНИЕ (править тут)
assets/js/products.js   ← ТОВАРЫ И ЦЕНЫ (править тут)
assets/js/app.js        логика: рендер каталога, корзина, оформление, скидка
assets/img/             фото товаров (slug-имена). Это собственные фото продавца

```
В корне также лежат исходники фото `photo_*.jpg` (бэкап оригиналов, сайтом не используются).

## Что где править (без знания кода)
- **Цены** — `products.js`, поля `price3 / price4 / price5` (обычная цена ДО скидки, в ₽).
  `0` → позиция покажется как «цена по запросу». Сейчас стоят ДЕМО-цены.
- **Скидка** — `config.js`, `discountPercent` (напр. `10`). `0` = выключить (исчезнут
  зачёркивания, значки «−N%» и верхняя плашка). Скидка считается автоматически, округляется до 10 ₽.
- **Контакты/реквизиты/название** — `config.js` (`shopName`, `phoneRaw`, `telegram`, `email`,
  `formspreeEndpoint`, объект `legal`, флаги `channels`).
- **Доставка** — массив `delivery` в `config.js`.

## Дизайн-система
- Тема: светлая, тёплый **персиковый градиент** фона, акцент — **золото** (`--gold`).
- Кнопки/тёмные элементы — тёплый **эспрессо** `--btn` (не чистый чёрный), задаётся в `:root`.
- Шрифты: **Cormorant Garamond** (заголовки), **Manrope** (текст), Google Fonts.
- Карточка: квадратное фото (`.card-media` 1:1) с тёплым фильтром на фото (sepia), бейджи
  бренда/пола, значок «−N%», описание+ноты раскрываются оверлеем `.card-hover` при наведении
  (на тач-устройствах — статично в `.card-mobile`). Блоки 3/4/5 мл `.vol-opt` — золотой акцент
  у активного, у цены активного блока лёгкий шиммер.
- Анимации: бегущая лента брендов, плавающее свечение в hero, появление по скроллу (`.reveal`→`.in`),
  раскрытие оверлея. Всё уважает `prefers-reduced-motion`.

## ВАЖНО: контракты JS — не ломать при правках разметки/стилей
`app.js` ищет элементы и вешает события по этим селекторам (можно менять оформление, НЕ удалять):

  footPhone, footWhatsapp, footTelegram, footEmail, promoBar, overlay, cartDrawer, cartClose,
  cartItems, cartEmpty, cartTotal, cartPriceHint, checkoutBtn, checkoutModal, checkoutClose,
  checkoutForm, checkoutTitle, formError, sendButtons`.
- Атрибуты: `[data-shop-name]`, `[data-shop-tagline]`; у инпутов формы `name="name|phone|address|
  comment|agreePdn|agreeOffer"`.
- Генерится в шаблонах `app.js` (если меняешь — сохрани классы/data): `.card[data-id]`, `.card-media`,
  `.vol-opt[data-vol]`+`.active`, `.add-btn`, `.card-badge/.card-gender-badge/.card-sale/.card-hover/
  .card-mobile`, цена `.p/.p-old/.p-now`, фильтр `.filter-chip[data-brand]`+`.active`,
  корзина `[data-act="inc|dec|rm"][data-key]`, кнопки отправки `[data-ch]` + `.send-wa/.send-tg/
  .send-email/.send-call`, `.reveal`/`.in`, `.toast`.
- НЕ менять структуру/имена полей в `products.js` и `config.js` и саму логику в `app.js`
  (только строки-шаблоны разметки).

## Ключевая логика (app.js)
- `let cart = loadCart()` читает `localStorage` на старте → корзина переживает перезагрузку.
- Цена: `priceFor(p,vol)` (3/4/5) → `effPrice` (со скидкой) → `priceTag(n)` (HTML: старая+новая).
  Корзина, итог и текст заказа считают по `effPrice`.
- Оформление: форма → текст заказа → отправка в WhatsApp (`wa.me`, префилл), Telegram (открыть
  чат + текст в буфер), Email (Formspree, иначе `mailto`), Обратный звонок.

## Запуск и проверка
- Локально: из папки проекта `py -3 -m http.server 8000` → http://localhost:8000
  (на Windows `python` — заглушка Store, использовать `py`).
- Синтаксис JS: `node --check assets/js/app.js`.
- Скриншоты (headless Chrome `"C:\Program Files\Google\Chrome\Application\chrome.exe"`):
  `--headless=new --no-sandbox --hide-scrollbars --force-device-scale-factor=1
  --user-data-dir=<СВЕЖАЯ_temp> --no-first-run --window-size=1280,900
  --screenshot=out.png --virtual-time-budget=6500 <url>`.
  Особенности headless: скриншот всегда от ВЕРХА (скролл игнорируется — для нижних секций
  делать временную копию с `<style>`, скрывающим верхние блоки); layout-viewport не опускается
  ниже ~484px (честный «телефон» так не снять, снимать при ширине 484); каждый запуск — со
  СВЕЖИМ `--user-data-dir` (иначе кэш). Временные копии писать UTF-8 БЕЗ BOM, иначе кириллица
  ломается. Открытую корзину/модалку снимать, засеяв `localStorage` ДО загрузки `app.js`
  (инъекция в `<head>`). Временные `__*.html` после съёмки удалять.
